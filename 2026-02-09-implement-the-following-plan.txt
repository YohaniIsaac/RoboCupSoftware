│ Contexto                                                                     │
│                                                                              │
│ El proceso de percepción PID envía frames (~921KB) al proceso de             │
│ visualización a través de multiprocessing.Pipe(). Cada envío serializa       │
│ (pickle) el numpy array completo, costando ~15-25ms por frame. A 30 FPS esto │
│  consume casi la mitad del tiempo disponible (33ms/frame), frenando la       │
│ percepción.                                                                  │
│                                                                              │
│ Solución                                                                     │
│                                                                              │
│ Reemplazar Pipe 2 (frame pipe) por Shared Memory para frames + Thread para   │
│ escritura asíncrona.                                                         │
│                                                                              │
│ Arquitectura                                                                 │
│                                                                              │
│ ANTES:                                                                       │
│   Percepción: [Detectar 5ms] → [Pickle+Send 921KB ~20ms] → [Detectar 5ms]    │
│   Ciclo: ~25ms = ~40 FPS                                                     │
│                                                                              │
│ DESPUÉS:                                                                     │
│   Percepción Main:    [Detectar 5ms] → [Queue.put 0.01ms] → [Detectar 5ms]   │
│   Percepción Thread:     [memcpy shared_mem 0.5ms] → [send metadata 0.2ms]   │
│   Ciclo: ~5ms (limitado por cámara)                                          │
│                                                                              │
